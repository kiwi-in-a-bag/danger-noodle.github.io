# 🐍 Danger Noodle Security
### Practical Compliance & SecOps Templates
**By Sean O'Hagan**

Welcome to the **Danger Noodle** vault. This is a personal project designed to host the compliance frameworks, security policies, and automation templates I've developed. The goal is to move away from "paper-only" compliance and toward documentation that actually works for Security Operations teams.

---

## 📂 Template Library

### 🛡️ Governance & Policy
* **[Information Security Policy (ISP)](./templates/isp-framework.md)** - A core framework for organizational security.
* **[Acceptable Use Policy (AUP)](./templates/aup.md)** - Clear guidelines for staff and system usage.
* **[Access Control Policy](./templates/access-control.md)** - Defining the principle of least privilege.

### 🔍 Risk & Audit
* **[Vendor Risk Assessment](./templates/vendor-assessment.md)** - Questionnaire for evaluating third-party security posture.
* **[ISO 27001:2022 Gap Analysis](./templates/iso-gap-check.xlsx)** - A simplified checklist for internal readiness.

### 🚨 Incident Response (IR)
* **[Ransomware Response Playbook](./templates/ir-ransomware.md)** - Step-by-step technical and communications guide.
* **[Post-Incident Review (PIR) Template](./templates/pir-template.md)** - Capture lessons learned and root cause analysis.

## 🛠️ The "Danger Noodle" Lab
This site is currently a mobile-first project developed while on leave. Once finalized, it will serve as the public front-end for my personal compliance library.

* **Host:** GitHub Pages (Transitioning to personal Debian/Proxmox lab later)
* **Philosophy:** "Compliance as Code"—simple, version-controlled, and accessible.
* **Format:** Markdown primary, with PDF/DOCX exports available.
